# Configuration

← [Back to README](../README.md)

Configuration is split between the stack environment file and the proxy runtime config:

- `infrastructure/.env`: dashboard, database, Docker, and deployment settings
- `infrastructure/config/config.yaml`: CLIProxyAPI runtime settings

## Environment Variables

### Core Stack Variables

| Variable | Required | Description | Example |
|----------|----------|-------------|---------|
| `DATABASE_URL` | Yes | PostgreSQL connection string used by the dashboard | `postgresql://cliproxyapi:pass@postgres:5432/cliproxyapi` |
| `POSTGRES_PASSWORD` | Yes | PostgreSQL password for the bundled database container | Generated by installer |
| `JWT_SECRET` | Yes | Secret for dashboard session cookies, minimum 32 chars | Generated by installer |
| `MANAGEMENT_API_KEY` | Yes | Shared secret used to call CLIProxyAPI management endpoints | Generated by installer |
| `INSTALL_DIR` | Yes | Absolute path to the repo root on the host | `/opt/cliproxyapi-dashboard` |
| `TZ` | No | Container timezone | `UTC` |

### Dashboard Variables

| Variable | Required | Description | Default |
|----------|----------|-------------|---------|
| `CLIPROXYAPI_MANAGEMENT_URL` | No | Internal management API URL | `http://cliproxyapi:8317/v0/management` |
| `DASHBOARD_URL` | No | Public dashboard URL used in links and notifications | `http://localhost:3000` |
| `API_URL` | No | Public CLIProxyAPI URL shown to users and clients | `http://localhost:8317` |
| `DOCKER_HOST` | No | Docker socket proxy URL for container management | `tcp://docker-proxy:2375` |
| `CLIPROXYAPI_CONTAINER_NAME` | No | Container name for the managed proxy service | `cliproxyapi` |
| `GITHUB_REPO` | No | GitHub repository used for dashboard update checks | `itsmylife44/cliproxyapi-dashboard` |
| `COLLECTOR_API_KEY` | No | API key used by the usage collector cron job | Generated by installer |
| `LOG_LEVEL` | No | Pino log level | `info` |
| `PROVIDER_ENCRYPTION_KEY` | No | 64 hex chars used to encrypt stored custom-provider API keys | — |

### Optional Webhook Variables

| Variable | Required | Description | Default |
|----------|----------|-------------|---------|
| `WEBHOOK_HOST` | No | URL of the optional deploy webhook service | `http://localhost:9000` |
| `DEPLOY_SECRET` | No | Shared secret for deploy webhook authentication | — |

## CLIProxyAPI Configuration

CLIProxyAPI runtime settings live in `infrastructure/config/config.yaml`.

The dashboard's **Proxy Settings** page edits the most common management-backed fields, including:

- `proxy-url`
- `request-retry`
- `max-retry-interval`
- `logging-to-file`
- `usage-statistics-enabled`
- `ws-auth`
- `force-model-prefix`
- quota exceeded handling
- routing strategy

You can still edit `config.yaml` directly for advanced cases, but the dashboard should be the primary control plane.

## Dashboard Settings

The **Settings** and **Admin** areas manage dashboard-local behavior such as:

- password changes
- session revocation
- max provider keys per user
- dashboard and proxy update operations

## Provider Management

The **Providers** page manages three kinds of upstream state:

- provider API keys
- provider OAuth accounts
- custom OpenAI-compatible providers

Custom providers can define:

- base URL
- API key
- optional prefix
- optional upstream proxy URL
- optional headers
- model alias mappings
- excluded model patterns

## Notes

- The bundled stack does not include a public reverse proxy.
- If you expose the dashboard or proxy publicly, terminate TLS and restrict access in your own reverse proxy or ingress layer.
